Shell Workshop

1. Welcome!

  • A shell is simply the outermost layer of an operating system, it's designed to provide a way for you to interact with the tools and servies that your operating system provides
  • One of the most widely used shells is called BASH, Mac and Linux computers use the BASH shell by default
  • The shell is the outermost layer of your operating system
  • CLI is a user interface where you give instructions to the computer by entering lines of text
  • GUI is a user interface where you give instructions to the computer by interacting with things like windows, menus, icons, and buttons

4. Your first command (echo)

  • echo in shell is similar to print and console.log in Python and JavaScript, respectively
  • !! is one way you could re-run your last command
  • Another way is to use the up and down arrow to cycle through recent commands
  • Use the left and right arrow keys to move the cursor

5. Variables

  • In BASH to assign values to variables use the = operator without spaces between the the variable name, operator, and value
  • To refer back to the variable, the dollar sign $ must be used in front of the variable name
  • The command line echo $COLUMNS x $LINES returns the size of the terminal window

6. Navigating directories (1/2)

  • ls is short for list, this command lists the contents of the current directory
  • cd stands for change directory, the command is use to move to another directory
  • BASH is a case sensitive language

7. Navigating directories (2/2)

  • pwd stands for print working directory
  • home directory is the current user's top-level directory (the directory that has all of the user's other directories inside of it
  • ls ~ will list the home directory regardless of current working directory
  • . for working directory
  • .. for parent directory
  • ~ for home directory

8. Options

  • -l stands for long as in long format, it gives a more detailed list than ls by itself
  • The -l option will give the date that each file was last modified, as well as the size of the file (in bytes)
  • * is used to recognize patterns, e.g. ls *pdf is used to pull a list of all the pdf files

9. Organizing your files (mkdir, mv)

  • mkdir is used to create a new directory
  • mv is used to mv a file
  • When you want to move files around on your computer, you have some different options - you could use your computer's graphical shell, or you could use the computer's text shell
  • Both do a lot of the same things, but in some cases using the text shell is way faster

10. Downloading (curl)

  • The curl command can be used to download a file from the Web
  • By default, it will download the file and display the contents in the terminal
  • CURL -> C URL -> see URL üòÇ
  • It is a good habit to put single quotes around the URL since a lot of URLs have special characters in them, such as the & sign, which have unusual meanings to the shell
  • If you add the -L option to curl, it will first follow any redirects, and then download the file from wherever the redirects ultimately go

11. Viewing files (cat, less)

  • The cat command will display the full contents of the file
  • cat is short for concatenate
  • The limitation of the cat command is that files often have more text than what will fit on the screen at once
  • The less command solves this issue by formatting it to show only what is able to fit on the screen at once
  • Once you've started the less program, you can use some different keys to move around
    • Spacebar or down arrow to scroll down
    • b key or up arrow to scroll up
    • Forward slash / to search
    • q key to exit less
