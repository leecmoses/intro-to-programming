18. Null, Undefined, and NaN

  • null is the "value of nothing", while undefined is the "absence of value"
  • NaN stands for "Not-A-Number" and it's often returned indicating an error with number operations

19. Equality

  • JavaScript is known as a loosely typed language due to its use of implicit type coercion
  • It's actually considered bad practice to use the == and != operators when comparing values for equality because of type coercion
  • In JavaScript it's better to use strict equality to see if numbers, strings, or booleans, etc. are identical in type and value without doing the type conversion first
