16. Truthy and Falsy

  • Every value in JS has an inherent boolean value, when that value is evaluated in the context of a boolean expression, the value will be transformed into that inherent boolean value
  • A value is falsy if it converts to false when evaluated in a boolean context
  • Here's the list of all of the falsy values:
    • the Boolean value false
    • the null type
    • the undefined type
    • the number 0
    • the empty string ""
    • the odd value NaN (stands for "not a number")
  • A value is truthy if it converts to true when evaluated in a boolean context
  • If it is not in the list of falsy values, then it's truthy!

17. Ternary Operator

  • Ternary operator provides a shortcut alternative for writing lengthy if...else statements
  • conditional ? (if condition is true) : (if condition is false)

19. Switch Statement

  • If you find yourself repeating else if statements in your code, where each condition is based on the same value, then it migt be time to use a switch statement
  • A switch statement is an another way to chain mulitple else if statements that are based on the same value without using conditional statements
  • The break statement can be used to terminate a switch statement and transfer control to the code following the terminated statement
    • By adding a break to each case clause, you fix the issue of the switch statement falling-through to other case clauses

20. Falling-through

  • In some situations, you might want to leverage the "falling-through" behavior of switch statements to your advantage
  • You can add a default case to a switch statement and it will be executed when none of the values match the value of the switch expression
  • Falling through should only be used in similar situations; otherwise, make sure to break each switch case to avoid unexpected behavior

7. Return Values

  • It's important to understand that return and print are not the same thing
    • console.log() - use to print a value to the JavaScript console
    • return - use to stop execution of a function and return a value back to the caller
  • Remember to only use console.log to test your code in the JS console
  • Once the code evaluates the first return statement, the function finishes

8. Using Return Values

  • A function's return value can be stored in a variable or reused throughout your program as a function argument

9. Scope

  • Scope refers to the visibility or accessibility of information

10. Scope Example

  • global scope - identifiers can be accessed everywhere within your program
  • function scope - identifiers can be accessed everywhere inside the function it was defined in

11. Shadowing

  • Shadowing can be avoided by declaring the variable both in the global and function scope
  • Completed exercises

12. Global Variables

  • It is important to minimize the use of global variables as much as possible because as programs get larger it gets harder to keep track and prevent conflicts between global variables

13. Scope Recap

  • If an identifier is declared in global scope, it's available everywhere
  • If an identifier is declared in function scope, it's available in the function it was declared in (even in functions declared inside the function)
  • When trying to access an identifier, the JavaScript Engine will first look in the current function, and it will continue to the next outer function until it reaches the global scope unless it finds the identifier before then

14. Hoisting

  • hoisting - before any JavaScript is executed, all function declarations are "hoisted" to the top of their current scope
  • To avoid bugs, JS programmers usually declare functions at the top of their scripts, and variables at the top of their functions
  • The variable declaration is hoisted to the top of current scope, but not the assignment

15. Hoisting Recap

  • Declare functions and variables at the top of your scripts, so the syntax and behavior are consistent with each other

16. Quiz: Build a Triangle (5-3)

  • Completed exercise

17. Function Expressions

  • Function expression is when a function is stored inside a variable
  • In a function expression the function no longer has a name because it's an anonymous function
  • All function declarations are hoisted and loaded before the script is actually run
  • All function expressions are not hoisted, since they involve variable assignment, and only variable declarations are hoisted, not the assigned value
  • The function expression will not be loaded until the interpreter reaches it in the script

18. Patterns with Function Expressions

  • Being able to store a function in a variable makes it really simple to pass the function into another function
  • A function that is passed into another function is called a callback
  • A function expression is when a function is assigned to a variable, in JS, this can also happen when you pass a function inline as an argument to another function
  • Anonymous inline function expressions are often used with function callbacks that are probably not going to be reused elsewhere, this is to save you from many lines of code

19. Function Expression Recap

  • Function Expression - when a function is assigned to a variable, it can be named or anonymous
  • Use the variable name to call a function defined in a function expression
  • You can even pass a function into another function inline, this pattern is commonly used in JavaScript, and can be helpful streamlining your code

5. Object Literals

  • Object-literal notation follows the following struturing:

  • Use camelCase for naming variables and objects
  • Names should not begin with a number

  • You do not need to wrap the key in quotes
  • If a key is a multi-word property, use camelCase
  • Do not use hyphens in property names

# The Document Object Model

1. Introduction

  • HTML and CSS are static, use JS to manipulate the DOM to make a website more dynamic

2. The DOM

  • The DOM stands for Document Object Model, it is the full, parsed representation of the HTML document, the relationship between elements, and contains the content and properties of the elements
  • The DOM is not part of the JavaScript language, the DOM is constructed from the browser and is globally accessible by JavaScript code using the document object

3. A CSS Mindset

  • Use the # at the beginning of the selector to target an element by its ID
  • Use the . (the period) at the beginning of the selector to target elements by their class

4. Select Page Element By ID

  • document.getElementById();
    • Need to pass the ID of the element that we want it to find and subsequently return to us
    • It already knows that we are searching for an ID, so the # is unnecessary
    • The method is called on the document object
    • it returns a single item
  • To read more about the method

5. Select Page Elements By Class or Tag

  • .getElementsByClassName();
  • .getElementsByTagName();
    • Both methods use the document object
    • Both returns multiple items
    • The list that's returned is not an array

6. Nodes, Elements, and Interfaces... Oh My!

  • characters -> tags -> tokens -> nodes -> DOM
  • Node = Class, node = object
  • Node (with a capital "N") is a blueprint that contains information about all of the properties and methods for real notes (with lowercase "n")
    • interface = blueprint
    • properties = data
    • methods = functionality
  • Just like the Node Interface, the Element Interface is a blueprint for creating elements
  • The Element Interface is a descendent of the Node Interface
  • .outerHTML is a property on the Element Interface
  • .id is a property on the Element Interface
  • .textContent is a property on the Node Interface
  • The Element Interface has its own .getElementsByClassName() that does the exact same thing as the one on the document object
    • This means you can use the document object to select an element, then you can call .getElementsByClassName() on that element to receive a list of elements with the class name that are descendents of that specific element!
  • list of Web API Interfaces

7. More Ways To Access Elements

  • Not all browsers support every standard, that's why almost every method on MDN has a Browser compatiability table that lists when each browser started supporting that specific method
  • jQuery has diminished in popularity over the years due to native DOM methods
  • document.querySelector();
    • returns a single element
  • document.querySelectorAll();
    • returns a list of elements
    • The list that is returned is a NodeList
    • It is possible to loop over a NodeList with either its .forEach() method, or a for loop
