27. Thinking about errors

  • There are three major kinds of errors that come up in programming: syntax errors, usage errors, and logic errors
  • Syntax errors are like spelling erros
  • When you're learning a new language, you'll probably make a lot of them, but once you're used to the language, you'll make very few
  • Usage errors are when you ask the computer to do something that doesn't make sense
  • Errors like usage errors are similar to saying something absurd, or giving a nonsense answer to a question
  • Python's NameError, Type Error and ZeroDivisionError messages usually mean an error of this sort
  • Logic errors is when the program runs fine, but it doesn't do what you wanted it to do, because something you wrote isn't what you meant
  • Indentation can be tricky, if you find that some lines of code are running more times (or fewer) than you expect, take a careful look at how your loops are indented
