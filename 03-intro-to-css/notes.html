Intro to CSS

1. Starting with style

  • CSS is used to change the color or the layout of a web page, among other things

2. Developer tools

  • Hotkey for dev tools is ALT + CMD + J
  • The view that you see in Elements is actually a picture of the way the browser thinks about the page internally - not just a copy of the original HTML file.

3. Text to trees

  • The particular tree structure the browser creates from HTML is called the DOM or Document Object Model

4. Tree Structure

  • Nodes are the different elements and the branches are the relationships between the nodes
  • The tree always has a root which is always the HTML element
  • Tree structure is not a diagram... it's a data structure - a way that information can be organized
  • Rules for trees
    1. A tree has a root node
    2. Each node can have branches that lead to other nodes (its children)
    3. Each node has only one parent (except the root)
  • DOM is the tree structure of HTML

14. Selectors: Class and ID

32. Color

  • Computers make colors by mixing light with the primary colors red, green, blue
  • To make gray, use equal amounts of red, green, and blue, but if all three colors are equal to zero, you'll get black; and if all three are the maximum value (255, FF, or 100%) you'll get white

33. Practice - Color

  • Higher values indicate brighter (more intense) light, and lower values indicate darker (less intense) light
  • Red, green, and blue light overlap to create secondary colors and white light
  • Hexadecimal is not as complex as it might appear, instead of having 9 digits, it has 16
  • Since there aren't 16 number symbols, Hexadecimal counts up to 9 and then starts using letters

35. Fonts

  • Generic font-family: serif, sans-serif, cursive, fantasy, and monospace
  • Font stack is a list of font alternatives; the browser will pick the first it can

38. Practice - Meaning vs. style

  • HTML code isn't only used by browsers that display on the screen, it is also used by search engines, smart speaker apps, and other programs
  • Thos programs can't see "boldface" or "italics", but they still need to know which text on a page is more important
  • HTML indicates what the contents mean, while the CSS indicates how the contents should look
  • Web programmers refer to this as semantic markup - using markup to indicate meaning, not just appearance
