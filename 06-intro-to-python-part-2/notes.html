4. Python in the terminal

  • The argument for the input function is optional, meaning that you can call the input function without passing it anything, input()

5. Interactive mode

  • Interactive mode is a great way to play around to see how Python works
  • Code that is written in interactive mode is not saved
  • CTRL + D Key or exit() function to quit interactive mode
  • The ... is showing you that you're constructing a compound statement, so whatever line you eneter next will be part of that statement
  • Unfortunately, if you make a mistake when defining a compound statement in interactive mode, you usually have to start over from the beginning, this is why interactive mode is generally only good for experimenting with short pieces of code
  • To get out of the loop and run the loop, you can simply hit Enter an extra time
  • The Python interpreter is the thing that actually reads and runs the code, when in interactive mode, the user is interacting with the interpreter

7. Functions in interactive mode

  • When a value is returned in interactive mode, the value gets displayed in the terminal

12. Methods on strings

  • Many string methods, like startswith and endswith are predicates they return True or False to indicate something about the string
  • isdigit returns a boolean for is this string made of only digits
  • islower returns a boolean for is this string made of only lowercase letters
  • isspace returns a boolean for is this string made of only whitespace characters, such as spaces and tabs

13. Truly boolean

  • For numbers, zero values are considered false, and nonzero vaules are considered true
  • For strings, the empty string is considered false, and all nonempty strings are considered true
  • For lists, the empty list is considered false, and all nonempty lists are considered true

14. Operations on lists

  • In Python, lists and strings are both sequence types - they represent a collection of values, not just a single value

15. Methods on lists

  • append adds one item to the end of a list
  • extend add possibly several items to the end of a list
    • The usual use of extend is with a list and another list not a list in the string
    • You could actually think of extend as a being a sort of append in a for loop
  • sort changes a list so it is in alphabetical (or numeric, etc.) order
  • reverse changes a list so it is in the opposite order from how it was before

16. Mutability and shared structure

  • Lists are mutable, while strings are immutable

17. Augmented assignments

  • An augmented assignment statement augments the existing value rather than replacing it
  • There are augmented assignment operators for all the arithmetic operators
  • The * operation on strings is used to repeat a string, you cannot multiply a string by a string, but you can "multiply" a string by an integer, to repeat a string some number of times
    • Doing this does not break the rule that strings are immutable, because it isn't altering the contents of the string - it's replacing the value of the variable

19. While loops

  • Linear search starts at the beginning of a sequence and looks at each item in the sequence until a match is found
  • A linear search stops either when a match is found or it runs out of items

20. Infinite loops and breaking out

  • Type CTRL + C to exit an infinite loop
  • A break statement will always skip to the end of the innermost while or for loop, if you have a loop inside another loop, it will only exit the inside loop
